#75 - Prevent the theater from calling 'close' twice if the animation delay and the escape button overlap
What & Why
The 1.8 s exit timer and the 3 s force-close button both call onClose() independently. In a stall scenario the 1.8 s timer fires onClose at 1.8 s (parent doesn't respond), then the user clicks force-close at 3 s and onClose fires a second time. The parent may not be idempotent on double close calls, leading to subtle state corruption.

Done looks like
onClose is wrapped in a once-guard inside TheaterRoom so the second call is a no-op regardless of which path fires first
The guard is reset when the theater re-mounts (each mount starts fresh)
Force-close button and the 1.8 s timer both remain in place — only the redundant second call is suppressed
